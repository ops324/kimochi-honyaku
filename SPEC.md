# きもち翻訳 — プロジェクト仕様書

最終更新: 2026年5月2日

---

## プロジェクト概要

**アプリ名**: きもち翻訳  
**コンセプト**: 「うまく言えない体や気持ちの状態」を、断片的なタップ入力からAIが翻訳し、親や医師、学校の先生に伝わる形に変換するアプリ。不調時だけでなく、調子がいい日や言葉にならない日の記録にも使える。

### 解決したい課題
- 子どもが「なんとなくしんどい」としか言えないとき、細かく説明させると本人の感覚から遠ざかる
- 正確な記録よりも「伝わること」がゴール
- 言語化・構造化しすぎると「説明される」になってしまう
- 単一の症状ではなく**組み合わせ**で意味を持つ不調（起立性調節障害など）を、子どもに難しい言葉を使わずに伝える
- **「しんどい / 元気」の二項対立**に当てはまらない日（なんとなく変、言葉にならない）も記録できること

### 設計の核心
- **翻訳の粒度**: AIがどこまで補完し、どこを曖昧なまま残すかのバランス
- **使われ続ける理由を丁寧に設計する**: 記録させるではなく、使いたくなる状態を作る
- **子ども側と大人側で語彙を分ける**: 子どもには優しい言葉、医師や学校には所見・配慮事項を伝える
- **記録しない自由を保障する**: 「今日は使わない」を選べることが継続の前提

## 画面構成（5画面）

### 画面0: ウェルカム（`#screen-welcome`）
- **時刻に応じたあいさつメッセージ**（5段階: 朝/昼/夕/夜/深夜、各2〜3バリアント）
  - 同じ時間帯でも日替わりでメッセージが変わる（`dayIndex % variants.length`）
- アプリアイコン（カスタムSVGゴーストキャラクター・浮遊アニメーション）
- グラデーションタイトル「きもち翻訳」
- **ストリークバッジ**（2回以上の記録で表示。localStorage管理）
  - 文言: 傾聴フレーム「あなたの声を ${n}回 きいてきたよ」（n≤6）
  - 長期継続バリアント（7回以上）: 回数ではなく在り続けた感を強調
  - 「X日連続達成」「記録継続中」などの義務感ある表現は使わない
- **CTAボタン**「今日のじぶんを ことばにする」
- サブテキスト「うまく言えなくても大丈夫」
- **脱出口ボタン**（ゴーストスタイル）「今日は記録しない」
  - タップ時: ボタンが消え、その場に「記録しない日も、ある。また来てね。」と3.2秒表示（`.soft-confirm`）
  - 責めず、次回を誘う一言で終わる

### 画面1: 入力（`#screen-input`）
- トップバー: 戻るボタン + 日付
- タイトル「今、どんな感じ？」
- **3択モード切り替えトグル**（`role="tablist"`、各ボタン`role="tab"`）
  - **しんどい感じ**（`#toggle-unwell`、デフォルト）
  - **うまく言えない**（`#toggle-unsure`）
  - **いい感じ**（`#toggle-good`）
  - 切り替え時: 選択状態リセット、CTAの活性条件を更新

- **カードグリッド選択**（3列グリッド、複数選択可）

  **しんどい感じモード（`#set-unwell`）**
  - **体のこと** [`data-category="body"`]: 頭が重い / だるい / めまいがする / 吐き気がある / 体が痛い / 食べられない
  - **ねむり** [`data-category="sleep"`]: 眠れなかった / 寝つけなかった / 夜中に起きた / 寝ても疲れがとれない / 怖い夢を見た
  - **気持ちのこと** [`data-category="mood"`]: 不安な感じ / 落ち込んでいる / ぼーっとする / 何もしたくない / イライラする / 泣きたい感じ
  - **状況のこと** [`data-category="situation"`]: 朝起きられない / 学校に行けない / 集中できない / 外に出たくない / 人と話したくない

  **うまく言えないモード（`#set-unsure`）**
  - [`data-category="unsure"`] の汎用カード4枚:
    - なんとなくしんどい / 言葉にならない / いつもと違う / もやもやする
  - **メモ欄のみでも翻訳可能**（カード0枚 + メモあり → CTAが活性化）
  - セクション末尾の補助テキスト（`.unsure-note`）:
    「カードを選ばなくてもいいよ。ことばにならない感じ、そのまま教えてね。」

  **いい感じモード（`#set-good`）**
  - **体のこと**: 体が軽い / 食欲がある / 頭がすっきり / 体に力がある / 気分がいい
  - **ねむり**: よく眠れた / すぐ眠れた / すっきり起きた / いい夢を見た
  - **気持ちのこと**: 楽しい気分 / 落ち着いている / やる気がある / 嬉しい感じ / ほっとしている / 安心している
  - **状況のこと**: 朝起きられた / 学校に行けた / 集中できた / 外に出られた / 人と話せた

- **客観値の任意手入力**（しんどい/いい感じ両モードで「ねむり」セクション末尾に表示）
  - **保護者向け補助フィールド**（`<details id="parent-panel">`、デフォルト閉じた状態）
  - 開閉テキスト: 「保護者の方へ — スマートウォッチなどで分かれば（任意）」
  - 前回入力した値がある場合はプレースホルダーに参考値を表示し、バッジ「前回の記録があります」を表示（`initParentPanelMemory()`）
  - ねむった時間: 0.5刻みの number input（h単位、空欄可）
  - 眠りの深さ: 5段階セグメントボタン（1=浅い〜5=深い）
  - 入力された値は医師・学校向け翻訳に同梱される
  - **子どものメインフローの最後（フォーム末尾）** に配置 → 子どもの選択体験を妨げない

- ひとことメモ（テキストエリア、任意）
  - うまく言えないモードではメモ入力でCTAが活性化する
- **CTA**「ことばに してもらう ✦」（1つ以上選択、またはうまく言えないモード時はメモ入力で活性化）
  - 非活性時のヒント: 「1つでも見つかったら、ことばにしますね。」
  - 活性化は `syncCTA()` が管理（モードとselected/memoを考慮）

### 画面2: 翻訳中（`#screen-loading`）
- ゴーストキャラクター（バウンス）+ 3色ドット
- 「あなたの状態を言葉にしています…」
- 所要時間: 2.2秒（APIコール想定）

### 画面3: 翻訳結果（`#screen-result`）
- **ムードカード**（ヒーロー要素・共有可能）
  - 色彩心理学に基づく配色（しんどい=ネイビーブルー / うまく言えない=モーブパープル / いい感じ=ゴールデンイエロー）
  - 翻訳テキスト（基本翻訳のみ。組み合わせ所見や時系列サマリは出さない）
  - タグ一覧（選択したカード）
- **フィードバックブロック**（`#feedback-block`）
  - 初期状態 `opacity: 0`、翻訳完了から **1.8秒後** に `opacity: 1`（`.appeared` クラス付与、540ms transition）
  - 狙い: 翻訳文を先に読む時間を確保し、判断の急かしを避ける（感情への共鳴余地を設ける）
  - 3択絵文字: ちがう / 近い / そのとおり
- **共有アクション**（4ボタン、2×2グリッド）
  - 親に見せる
  - 医師に渡す
  - 学校に渡す（保健室・先生向け）
  - 画像で保存（プレースホルダー）
- 「きょうは ここまで」ゴーストボタン（結果画面を閉じてウェルカムへ戻る）

### 画面4: 共有プレビュー（`#screen-share`）
- モード: **親向け / 医師向け / 学校向け** で文面・アイコン・含まれる情報が変わる
- ドキュメント型カード + 「テキストをコピー」ボタン
- コピー時のヘッダー:
  - 親向け: 「【親への共有メモ】」
  - 医師向け: 「【受診時の参考メモ】」
  - 学校向け: 「【学校への共有メモ】」

## AIロジック（現在: モック）

### 翻訳パイプの2段階

```
[選択タグ + メモ + 客観睡眠値 + モード]
   ↓
1. translate(tags, memo, mode)  — 子ども・親に見せる本文
   ↓
2. formatForAudience(audience, ctx) — 提出先別に最終文面を生成
   ↓
[親向け / 医師向け / 学校向け]
```

### 1. translate (基本翻訳)

選択されたタグをカテゴリ別・モード別に自然な日本語文に変換する。

**しんどい感じモード** (`mode === 'unwell'`):
```
[体] + [ねむり] + [気持ち] + [状況] + [メモ] → 1〜4文の要約
```
接続詞: 体「。」→ ねむり「また、」→ 気持ち「また、」→ 状況「そのため、」→ メモ「本人からは「〜」とのこと。」

**うまく言えないモード** (`mode === 'unsure'`):
- UNSURE_MAP（4枚のカード用の短文フレーズ）から断片を拾い、「なんとなくいつもと違う感じがある」を基底として繋ぐ
- カード0枚 + メモのみの場合:「今日は言葉にならない何かがある。本人いわく「〜」とのこと。」
- いずれも「言語化を強いない」文体を保つ

**いい感じモード** (`mode === 'good'`):
- 接続詞を「さらに、〜」中心にして肯定的に。

### 2. 組み合わせシグナル検出 (detectSignals)

選択タグの組み合わせを **ルールベース** で判定し、医師・学校向け翻訳に「考えられるパターン」として注記する。**親向け・子ども向けには出さない**。

| ID | 名称 | マッチ条件 |
|---|---|---|
| `od` | 起立性調節障害（OD）の典型像 | `朝起きられない` ∧ `めまいがする` ∧ (`頭が重い` ∨ `だるい`) |
| `headache` | 頭痛発作の示唆 | `頭が重い` ∧ (`吐き気がある` ∨ `めまいがする`) |
| `circadian` | 概日リズム睡眠の乱れ | `朝起きられない` ∧ (`寝つけなかった` ∨ `夜中に起きた` ∨ `眠れなかった`) |
| `anxiety` | 不安・緊張反応の示唆 | `不安な感じ` ∧（`寝つけなかった`・`食べられない`・`泣きたい感じ` のうち2つ以上） |
| `low_energy` | エネルギー低下の遷延 | `だるい` ∧ `落ち込んでいる` ∧ (`何もしたくない` ∨ `食べられない`) |

優先度は **OD > headache > circadian > anxiety > low_energy**、最大2件まで注記する。確定診断は匂わせず「典型像と類似する組み合わせ」と表現する。

うまく言えないモードではシグナル検出をスキップする（カード4枚の意味が具体的症状でないため）。

### 3. 提出先別フォーマッタ (formatForAudience)

| audience | アイコン | トーン | 含める要素 |
|---|---|---|---|
| `parent` | 👨‍👩‍👧 | 子ども目線・優しい | 翻訳本文 + タグ（睡眠は「短め/浅め」等のソフト表現） |
| `doctor` | 🏥 | 医学的SOAP風 | **S**: 本人の訴え / **O**: 選択タグ・睡眠客観値 / **A**: 組み合わせ所見 / **P**: 継続観察と最近30日の傾向サマリ |
| `school` | 🏫 | 配慮事項中心・病名暗示控えめ | 教室での配慮事項 + 直近の登校状況 + 【背景にある発達特性】節 |

**うまく言えないモードでの各audience処理**:
- `parent`: 「今日は言葉にしにくい状態でした」として翻訳本文を添える
- `doctor`: SOAP【A】に「非特異的主観的愁訴（non-specific subjective complaint）」と記載。特定シグナルへの帰属は行わない
- `school`: 「体調が優れず、言語化しにくい状態です」「無理に詳しく話させず、選択肢を提示してください」を推奨事項として記載

**学校向けの【背景にある発達特性】節**（思春期・概日リズム関連シグナルがある場合のみ）:
- 病名は出さず「思春期の体内時計の変化」として説明
- 根拠: 志村哲祥『寝た子は起こすな』NHK出版, 2026（思春期の睡眠相後退は生物学的特性）

### 4. 共有テキスト生成

```
【親への共有メモ / 受診時の参考メモ / 学校への共有メモ】
YYYY年MM月DD日（曜）

{formatForAudience の結果}

記録されたこと：{タグ一覧}

きもち翻訳より
```

## 状態管理（app.js）

```js
const state = {
  selected:     Set,                       // 選択中のカード（JSON文字列）
  memo:         string,                    // メモテキスト
  translation:  string,                    // 基本翻訳結果
  tags:         Array,                     // { val, cat } の配列
  shareMode:    string,                    // 'parent' | 'doctor' | 'school'
  feedbackDone: boolean,                   // フィードバック済みフラグ
  mode:         string,                    // 'unwell' | 'unsure' | 'good'
  sleepHours:   number | null,             // 客観睡眠時間（任意）
  sleepScore:   1 | 2 | 3 | 4 | 5 | null, // 眠りの深さ（任意）
  signals:      string[],                  // detectSignals の結果（id配列）
}
```

### ストリーク管理（localStorage）
- `kmt_streak`: 累計記録回数（数値）
- `kmt_last`: 最終記録日（YYYY-M-D 文字列）
- 翻訳実行時に `bumpStreak()` を呼び出し
- 2回以上でウェルカム画面にバッジ表示
- 文言: 傾聴フレーム（「声を聞いてきた」視点）— 義務感・プレッシャーを与えない

### 記録履歴の永続化（localStorage）
- `kmt_records`: 過去の記録の配列（直近180日まで保持）

```js
// kmt_records の各要素
{
  id: number,                         // Date.now()
  date: 'YYYY-MM-DD',
  mode: 'unwell' | 'unsure' | 'good',
  tags: [{ val, cat }],
  memo: string,
  sleepHours: number | null,
  sleepScore: number | null,
  signals: string[],                  // unsure モード時は []
  translation: string,
}
```

### 傾向サマリ (summarizeRecent)

直近N日の記録を集計して医師・学校向け翻訳に同梱する。

```js
summarizeRecent(days = 30) → {
  totalDays,            // 期間中の記録日数
  unwellDays,           // しんどいモードだった日数
  topTags,              // 頻出タグ Top3（unsureタグは除く）
  morningStruggleCount, // '朝起きられない' 出現日数
  sleepHoursMedian,     // 睡眠時間の中央値（客観値）
  recurringSignals,     // 同じ signal の繰返し出現
}
```

### 保護者パネルの前回値記憶 (initParentPanelMemory)

`kmt_records` を後ろから走査し、`sleepHours` または `sleepScore` がある最新レコードを探す。
見つかった場合:
- `sleepHours` input の placeholder に「前回: X.Xh」を表示
- `sleepScore` セグメントに前回値をデフォルト選択
- パネル summary 要素に `.parent-panel-memo`（バッジ）を追加

## デザインシステム

### カラーパレット（デザイントークン）

| トークン | 値 | 用途 |
|---|---|---|
| `--bg` | `#FBF8F4` | ページ背景（クリーム） |
| `--surface` | `#FFFFFF` | カード背景 |
| `--border` | `#EDE8E1` | 境界線 |
| `--ink-1` | `#1E1B2E` | 本文テキスト |
| `--ink-2` | `#6B6478` | サブテキスト |
| `--ink-3` | `#ADA6B8` | ヒント・プレースホルダー |
| `--brand` | `#7C6FE0` | ブランドカラー |
| `--grad-brand` | `#7C6FE0 → #B39DFA → #F9A8C9` | ボタン・グラデーション |

### ムードカード配色（色彩心理学に基づく）

| モード | 色 | 根拠 |
|---|---|---|
| しんどい感じ | `#060A12 → #1E5EAE`（ネイビーブルー） | Jonauskaite et al. 2020 — 憂鬱感・不快感との色-感情対応 |
| うまく言えない | モーブパープル（`#EDE9F2` 系） | 非言語・曖昧・霧感のニュアンス |
| いい感じ | `#FFFBEB → #F59E0B`（ゴールデンイエロー） | 128年分メタ分析 2024 — 喜び・活力との対応 |

### カードデザイン（シール風）

**SVGイラスト**: 全35枚+、カテゴリ別絵文字ベースのアイコン（44×44px → 50px レンダリング）

**シール風スタイル**:
- 各カードは `nth-child` パターンで −1.6° 〜 +1.8° のランダムな傾きを持つ（6パターン）
- 選択時: 傾きが増幅され、さらに動いた感を出す
- ラベル: `font-size: 12.5px / font-weight: 800 / color: var(--ink-1)`

**カード別カラートークン**:
各カードは `[data-value="XXX"]` セレクタで `--card-tint`（ライトカラー）と `--card-border`（ボーダーアクセント）を持つ。
カード背景: `linear-gradient(180deg, var(--card-tint) 0%, #FFFFFF 100%)`

カテゴリ別のカラーパレット方針:
- 体（body/unwell）: 赤橙〜青紫系（症状の色彩心理学に準拠）
- 睡眠（sleep/unwell）: 藍〜深紫系（夜・休息のイメージ）
- 気持ち（mood/unwell）: ピンク〜青灰系（感情スペクトラム）
- 状況（situation/unwell）: 緑灰〜茶系（外的状況感）
- 言えない（unsure）: フォグ・モーブ統一（`--card-tint: #EDE9F2`）
- 体/睡眠/気持ち/状況（good系）: 黄緑〜サニー系

### ムードカードのアンビエントドリフト（ambient drift）

3D チルト（JS + DeviceOrientation）は **完全撤去**。  
代替として CSS `@keyframes ambient-drift` による18秒ループのライト拡散アニメーション。

```css
.mood-card::before {
  content: '';
  position: absolute;
  /* 半径 200px の光源が 5ウェイポイントをゆっくり移動 */
  animation: ambient-drift 18s ease-in-out infinite;
}
@keyframes ambient-drift {
  0%, 100% { transform: translate(-60px, -60px); }
  25%       { transform: translate(80px, -40px); }
  50%       { transform: translate(60px, 80px); }
  75%       { transform: translate(-80px, 60px); }
}
```

ホバー時: `translateY(-3px)` の控えめなリフトのみ（チルトなし）。

### フィードバックブロックの遅延表示

```css
#feedback-block        { opacity: 0; transition: opacity 540ms ease; }
#feedback-block.appeared { opacity: 1; }
```

翻訳完了から 1800ms 後に JS が `.appeared` クラスを付与。  
理由: 翻訳文を先に読む時間を確保し、感情への共鳴余地を作る（チェックリスト化を防ぐ）。

### アクセシビリティ

- 全ての操作ボタンに `aria-label` を付与
- モード切り替えトグルは `role="tablist"` / `role="tab"` / `aria-selected`
- フォーカス時の輪郭: `focus-visible` のみ（マウス操作では非表示）、`--brand` カラーの2pxアウトライン
- `prefers-reduced-motion: reduce` メディアクエリで全アニメーション無効化（ambient-drift / particles / bounce / ghost-float すべて）

## 設計ポリシー

### UI/UX 心理学原則（本アプリへの適用）

1. **認知負荷の最小化**: カードは「選ぶだけ」。タップ1回で意思表示が完結する。
2. **感情への共鳴余地**: フィードバックを1.8秒遅らせ、翻訳文を読む時間を先に与える。チェックリスト的処理を避ける。
3. **記録の圧力をゼロにする**: 「今日は記録しない」ボタンを最初の画面から提供。達成感より「聴いてきた」という傾聴フレームにストリークを変換。
4. **二項対立の緩和**: 「しんどい / いい感じ」の2択に収まらない日のための第3モード。カード0枚でもメモだけで翻訳できる。
5. **親子の分離**: 保護者向け客観値フィールドはフォーム末尾の折りたたみセクションに配置し、子どものメインフローに割り込まない。
6. **前回記憶による心理的安全**: 保護者パネルに前回入力値を参照表示することで「また一から入力」の摩擦を軽減。
7. **病名暗示の制御**: 学校向け文面には配慮事項のみ、医師向けも「典型像と類似」止まり。過度な医療化を避ける。

### 言語・文体ポリシー

- 子どもへの語りかけは平仮名多用・体言止め禁止・命令形禁止
- 共有文面の医学的表現は医師向けのみ。親・学校向けは「眠りの様子: 短め」等のソフト表現
- ストリーク文言: 義務感（「達成」「継続」）ではなく傾聴感（「声を聞いてきた」）

### データアーキテクチャポリシー

- **子ども側の語彙を変えない**: シグナル・所見・サマリは医師/学校向け文面でのみ表面化
- **客観値はあくまで補助**: 主観カードを否定しない。「客観X時間 / 主観: ◯◯」と並列表示
- **病名は暗示まで**: 学校向けは「教室での配慮」、医師向けも「典型像と類似」
- **保護者の手入力で受ける**: HealthKit / Health Connect の直接連携はWeb制約上見送り
- **うまく言えないモードのシグナル検出はスキップ**: 汎用カードへのシグナル帰属は誤検出リスクが高い

## 参考文献・着想元

設計判断の根拠として参照した一次資料を以下に記録する。実装に直接根拠を持たせるためのものであり、医療的判断の代替ではない。

- **志村哲祥『寝た子は起こすな「早起き神話」の深刻な現実』NHK出版, 2026年4月10日, 200ページ**
  - 著者は東京医科大学睡眠学講座客員教授、スタンフォード大学で睡眠研究を実施した経歴を持つ。
  - 第5章「病気としての『朝起きられない』」、第4章「概日リズムの睡眠への影響」が、本アプリの主軸ターゲット（OD・概日リズム障害）と直接対応。
  - 学校向け文面の【背景にある発達特性】節（思春期の体内時計後ろずれの説明）の根拠としている。
  - シンギュラボメンバーからの情報提供。

- **株式会社こどもみらい (https://cfltd.co.jp)**
  - STRESCOPE / eSLEEP / カラダスコープ / 学校保健の事業領域に着想を得て、  
    症状の組み合わせシグナル・睡眠カテゴリの独立化・時系列傾向サマリ・提出先別トーン切替を設計。
  - シンギュラボメンバーからの情報提供。

- **色彩心理学資料**
  - Jonauskaite, D. et al. (2020). Universal patterns in color-emotion associations. *Psychological Science*.（憂鬱感とネイビーブルーの対応）
  - Meta-analysis on color-emotion (2024).（喜び・活力とゴールデンイエローの対応、128年分のデータ）
